# User Stories

Anforderungen im **Scholar-Centered Design** Format.

## 1. Funktionale Anforderungen

- **US 1:** Als **Musikwissenschaftler:in** möchte ich die Intensität von Affekten visualisieren, um die dramaturgische Kurve einer Szene zu verstehen.
- **US 2:** Als **Forscher**:in möchte ich die KI-Begründung einsehen, um die Validität der automatischen Analyse hermeneutisch zu prüfen.
- US 3: Als Forscher:in möchte ich, dass unklare Textstellen (Ironie, OCR-Fehler) visuell hervorgehoben werden, damit ich sie manuell prüfen kann.

## 2. MoSCoW-Priorisierung

- **Must-have:** JSON-Extraktion der Affekte, Historische Begründung.
    
- **Should-have:** Export-Funktion der Ergebnisse.
    
- **Could-have:** Grafische Darstellung der Affekt-Kurve.
